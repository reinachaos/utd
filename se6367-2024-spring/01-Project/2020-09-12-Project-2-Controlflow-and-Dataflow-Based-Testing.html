Project 2: Controlflow- and Dataflow-Based Testing

Part 1

(70 points)

Use xSuite to measure the block, decision, def-use coverage on the Unix “spell” program with respect to the test cases which you submitted for the functional testing (i.e., test cases created for your Project 1).

Instrument the subject program

The instrumentation can be done by running:

make clean
make CC="atac cc"

Recall what you did in xSuite tutorial and use xSuite to measure the code coverage of your test cases.

Submission for part 1

You need to submit:

  • a tar (or zip) file, and name it to Project-2A-FirstName-LastName.tar with the following:
    • your entire project 2 folder (including all the files and scripts to run the test cases you created). If a special setup is required for some of your test cases, you need to schedule an appointment with the TA to do this before the deadline.
    • a coverage summary (screenshot) similar to Figure 2-22 in Chapter 2 of the xSuite User’s Manual

(Submissions without the correct file name will not be accepted.)

Part 2

(150 points)

Generate additional test cases based on the hints provided by xSuite or
other information which you have to improve the block and decision coverage.

Submission for part 2

Submit the following required items to eLearning.

  • a tar (or zip) file, and name it to Project-2B-FirstName-LastName.tar with the following:

    • download the template, and enter your answers:

      • the 1st column is a test case ID starting from T001

      • the 2nd column is a description of this test case

      • the 3rd column explains the technique(s) used for generating this test case (e.g.,reading the man page, equivalence partitioning, boundary value analysis, predicate testing, using hints provided by xSuite, etc.)

      • the 4th column is the amount of time (in minutes) you spent on creating this test case

      • the 5th column is the cumulative block coverage
        (the cumulative coverage on each row should be the coverage after executing the test case of that row. If you have already loaded all the test cases in xSuite, you can obtain this information by right click any test cases and select “Disable all tests”. Then right click each test case and select “Enable this test”. Write down the cumulative coverage reported at “Total”. Repeat this process until all the test cases are enabled)

      • the 6th column is the cumulative decision coverage
        (the cumulative coverage on each row should be the coverage after executing the test case of that row. Please use the similar approach explain for the 5th column to obtain this information.)

      (Note: test cases in this table needs to be a superset of those submitted for Project 1)

    • your entire project 2 folder (including all the files and scripts to run the test cases you created). Compared to the Part 1, the project folder you submit should include additional test cases. If a special setup is required for some of your test cases, you need to schedule an appointment with the TA to do this before the deadline.

(Submissions without the correct file name will not be accepted.)
