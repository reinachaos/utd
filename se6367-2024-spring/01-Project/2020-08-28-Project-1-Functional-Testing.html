Project 1: Functional Testing

(Total points: 100)

In this project, you are required to generate a test set for testing the Unix “spell” command. The objective is to conduct good functional testing to conform to all the functional specifications.

Login to the project server

Follow the instructions on our website to connect to the project server.

Read the manual of the subject program

You can review the man page of “spell” by running the spell program with the help option:

 ./spell -h

Execute some test cases

Run spell against a sample test case using the following command:

./spell sample

Run spell against some test cases you created

Project submission

Submit a tar (or zip) file (other file extensions will not be accepted)(Project-1-FirstName-LastName.tar) of the following three files to eLearning.

(Files do not follow the required naming rules will not be accepted)

  • Functionality Table
    Download the template on our website and enter your answer. The functionality table has two columns:

    • The first column is a functionality ID starting from F01.
    • The second column is a description of the functionality

  • Test Case Table
    Download the template on our website and enter your answer. The test case table has four columns:

    • The first column is a test case ID starting from T001
    • The second column is a description of this test case (i.e., its objective)
    • the third column indicates which functionalities are tested by this test case
      (i.e., listing the corresponding functionality IDs)
    • the fourth column explains the technique used for generating this test case
      (e.g., reading the man page, equivalence class partitioning, boundary value analysis, predicate testing, etc.)

  • A Shell Script
    A shell-script “Your-LastName-Test-Execution.sh” which can be used to automatically execute all the test cases you submitted. The output of each execution should be saved in “Your-LastName-Test.Output” and the errors, if any, should be saved in “Your-LastName-Test.Error”.

Note: if your shell script requires additional files to run (e.g., you created some files to be used as program inputs), you can tar and submit the entire project folder, including the two required Excel sheets
